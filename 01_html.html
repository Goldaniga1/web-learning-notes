01 — HTML 基礎

網頁的骨架 · 瀏覽器讀的語言

HTML 是什麼？

HTML（HyperText Markup Language）是用來描述網頁內容結構的語言。它告訴瀏覽器「這是標題」「這是段落」「這是圖片」，但不管顏色或動畫——那是 CSS 和 JavaScript 的事。

🏠 比喻：HTML = 房子的牆壁和樓板（結構）／CSS = 油漆和裝潢（外觀）／JS = 電燈開關和電梯（互動）

每個 HTML 檔案的基本結構

<!-- 告訴瀏覽器：這是 HTML5 文件 --> <!DOCTYPE html> <html lang="zh-TW"> <head> <!-- 給瀏覽器看的資訊，不顯示在畫面上 --> <meta charset="UTF-8"> <!-- 支援中文 --> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <!-- 手機支援 --> <title>頁面標題（出現在分頁）</title> </head> <body> <!-- 畫面上看得到的東西都在這裡 --> <h1>我是標題</h1> <p>我是段落</p> </body> </html>
規則：大部分標籤都有開標籤 <p> 和關標籤 </p>。少數例外不需要關標籤（叫 void element）：<img>、<input>、<br>、<meta>

最常用的標籤

文字類

標籤用途例子
<h1> ~ <h6>標題，h1 最大，h6 最小文章標題、章節標題
<p>段落一般文章內容
<strong>粗體（語意：重要）關鍵字
<em>斜體（語意：強調）書名、術語
<br>換行地址分行
<span>行內容器（沒有語意）部分文字加樣式

結構類（語意元素）

標籤代表意義
<header>頁首區域
<nav>導覽列
<main>主要內容
<section>一個獨立區塊（有主題）
<article>獨立文章
<footer>頁尾
<div>通用容器（沒有語意，用來排版）
語意元素 vs div：用 <nav> 而不是 <div id="nav">，讓程式碼更清楚，也讓搜尋引擎和螢幕閱讀器更容易理解頁面。SDG 11 專案就是這樣寫的。

屬性（Attributes）

標籤可以加屬性來補充資訊，格式是 屬性名="值"，寫在開標籤裡面。

<!-- 連結：href 決定要去哪，target="_blank" 表示新分頁開 --> <a href="https://example.com" target="_blank">點我</a> <!-- 圖片：src 是圖片路徑，alt 是圖片讀不出來時顯示的文字 --> <img src="images/photo.jpg" alt="城市夜景" width="400"> <!-- id 是唯一識別（一頁只能有一個同樣的 id）--> <div id="hero">...</div> <!-- class 可以重複用，多個元素共用同一個樣式 --> <div class="card">卡片 A</div> <div class="card">卡片 B</div> <!-- data- 屬性：自訂資料，SDG11 用來存中英文 --> <span data-en="Statistics" data-zh="統計數據">Statistics</span>

id vs class — 最常搞混的概念

✅ id — 唯一，一個就好
<nav id="main-nav">...</nav>
<section id="hero">...</section>
<footer id="footer">...</footer>

用途：
- CSS: #hero { ... }
- JS: document.getElementById('hero')
✅ class — 可重複，多個元素共用
<div class="card">台北</div>
<div class="card">高雄</div>
<div class="card">台中</div>

用途：
- CSS: .card { ... }
- JS: querySelectorAll('.card')

表格（Table）

SDG 11 的「城市案例」區塊就是用 table 做的。

<table> <thead> <!-- 表頭 --> <tr> <!-- tr = table row 一列 --> <th>城市</th> <!-- th = table header 粗體標題欄 --> <th>國家</th> </tr> </thead> <tbody> <!-- 表格內容 --> <tr> <td>台北</td> <!-- td = table data 一格 --> <td>台灣</td> </tr> <tr> <td>哥本哈根</td> <td>丹麥</td> </tr> </tbody> </table>

列表

<!-- 無序列表（圓點）--> <ul> <li>台北</li> <li>高雄</li> </ul> <!-- 有序列表（數字）--> <ol> <li>第一步</li> <li>第二步</li> </ol> <!-- nav 的導覽列就是用 ul + li 做的 --> <nav> <ul> <li><a href="#stats">Statistics</a></li> <li><a href="#taiwan">Taiwan Focus</a></li> </ul> </nav>

表單（Form）

<form action="/submit" method="post"> <!-- label 的 for 要對應 input 的 id --> <label for="name">姓名</label> <input type="text" id="name" name="name" placeholder="請輸入姓名"> <!-- 各種 type --> <input type="email" placeholder="電子郵件"> <input type="password" placeholder="密碼"> <input type="number" min="0" max="100"> <input type="checkbox"> 我同意條款 <input type="radio" name="city" value="taipei"> 台北 <!-- 下拉選單 --> <select> <option value="taipei">台北</option> <option value="kaohsiung">高雄</option> </select> <!-- 送出 / 重設按鈕 --> <input type="submit" value="送出"> <button type="button" onclick="doSomething()">自訂按鈕</button> </form>

巢狀結構（Nesting）

HTML 標籤可以互相包含，形成層層巢狀的結構。SDG 11 的城市卡片就是這樣：

<!-- 城市卡片的實際結構 --> <div class="tw-city-card"> <!-- 最外層：整個卡片 --> <h3> <div class="city-icon">北</div> <!-- 圓形圖示 --> <span>Taipei</span> <!-- 城市名稱 --> </h3> <div class="progress-item"> <!-- 一條進度條 --> <div class="progress-label"> <span>Recycling Rate</span> <span>55%</span> </div> <div class="progress-track"> <div class="progress-fill" data-value="55"></div> </div> </div> </div>

路徑寫法（重要！）

情境路徑寫法
同資料夾的圖片src="photo.jpg"
子資料夾src="images/photo.jpg"
上一層資料夾src="../photo.jpg"
外部網址src="https://example.com/photo.jpg"
頁面內跳轉href="#section-id"
SDG11 的 images/新竹2.png 就是「子資料夾」的寫法。HTML 檔案在 sdg11/，圖片在 sdg11/images/，所以路徑寫 images/新竹2.png。

注解（Comment）

<!-- 這是注解，瀏覽器不會顯示，給自己或隊友看的 --> <!-- Hero Section --> <section id="hero"> ... </section> <!-- Stats Section --> <section id="stats"> ... </section>
快捷鍵：在 VS Code 選取程式碼後按 Ctrl + / 可以快速加上/移除注解。
