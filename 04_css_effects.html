04 — CSS 進階效果

transition · animation · CSS 變數 · media query · backdrop-filter

transition — 平滑過渡效果

當屬性改變（如 hover）時，不要瞬間跳變，而是平滑地過渡。

.btn { background: #f99d26; transform: translateY(0); /* transition: 屬性 時間 時間函數 */ transition: transform 0.2s ease, box-shadow 0.2s ease; /* 也可以寫 all，讓所有屬性都有過渡 */ /* transition: all 0.3s ease; */ } .btn:hover { transform: translateY(-4px); /* hover 時往上移 4px */ box-shadow: 0 8px 24px rgba(249,157,38,0.4); }

常用的 transition timing function（時間函數）

transition: all 0.3s ease; /* 開始結束都慢（最自然）*/ transition: all 0.3s linear; /* 均速 */ transition: all 0.3s ease-in; /* 開始慢，結束快 */ transition: all 0.3s ease-out; /* 開始快，結束慢（滑入感）*/ transition: all 1.3s cubic-bezier(0.4, 0, 0.2, 1); /* 自訂曲線（SDG11 進度條）*/

transform — 變形

transform: translateY(-7px); /* 往上移 7px（不影響其他元素！）*/ transform: translateX(20px); /* 往右移 */ transform: scale(1.05); /* 放大 5% */ transform: rotate(45deg); /* 旋轉 45 度 */ /* 組合使用 */ transform: translateY(-50%) scale(1.1);
為什麼用 transform 而不是 margin？
transform 移動不會影響其他元素的位置，且由 GPU 加速，動畫更順。SDG11 的卡片 hover 效果都用 translateY 而不是改 margin-top。

@keyframes — 關鍵影格動畫

transition 是「A → B 平滑」，keyframes 是「循環播放的動畫」。

/* 定義動畫 */ @keyframes bounce { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-12px); } } /* 套用在元素上 */ .scroll-arrow { animation: bounce 2s infinite; /* 名稱 時長 循環次數 */ } /* 進場動畫（SDG11 fade-up 效果的概念）*/ @keyframes fadeUp { from { opacity: 0; transform: translateY(36px); } to { opacity: 1; transform: translateY(0); } } .card { animation: fadeUp 0.65s ease forwards; /* forwards = 停在最後一幀 */ } /* 延遲動畫（讓卡片一張一張進場）*/ .card:nth-child(2) { animation-delay: 0.1s; } .card:nth-child(3) { animation-delay: 0.2s; }

@media — 媒體查詢（手機版）

根據螢幕寬度套用不同的 CSS，這就是讓網頁自動適應手機的方式。

/* 螢幕寬度 ≤ 768px 時（手機版）*/ @media (max-width: 768px) { nav ul { display: none; } /* 隱藏導覽列 */ .nav-hamburger { display: flex; } /* 顯示漢堡按鈕 */ #hero h1 { font-size: 2rem; } /* 標題字縮小 */ section { padding: 60px 20px; } /* 左右 padding 縮小 */ .tw-city-card { width: 100%; } /* 卡片全寬 */ } /* 也可以用 min-width（從手機設計，往大螢幕加）*/ @media (min-width: 769px) { /* 平板和桌機的樣式 */ } /* 常見斷點 */ /* 手機：max-width: 480px */ /* 平板：max-width: 768px */ /* 桌機：min-width: 1024px */

backdrop-filter — 毛玻璃效果

讓元素背後的內容模糊，做出玻璃質感。SDG11 導覽列和 Modal 都有用。

nav { background: rgba(249, 157, 38, 0.88); /* 半透明背景 */ backdrop-filter: blur(14px); /* 背後模糊 */ -webkit-backdrop-filter: blur(14px); /* Safari 需要加前綴 */ }

漸層（Gradient）

/* 線性漸層 */ background: linear-gradient(135deg, #0f0c29, #302b63, #24243e); /* 角度 顏色1 顏色2 顏色3 */ /* 由上到下 */ background: linear-gradient(to bottom, rgba(0,0,0,0.8), rgba(0,0,0,0)); /* SDG11 場景 Modal 底部遮罩 */ background: linear-gradient( to top, rgba(0,0,0,0.88) 0%, /* 底部很暗 */ rgba(0,0,0,0.35) 55%, /* 中間半透明 */ rgba(0,0,0,0.18) 100% /* 頂部幾乎透明 */ );

box-shadow — 陰影

/* box-shadow: X偏移 Y偏移 模糊 擴散 顏色 */ box-shadow: 0 4px 24px rgba(0,0,0,0.08); /* 輕柔陰影 */ box-shadow: 0 8px 32px rgba(249,157,38,0.4); /* 橘色光暈 */ box-shadow: 0 2px 12px rgba(0,0,0,0.5); /* 文字陰影 */ /* text-shadow 給文字加陰影 */ text-shadow: 0 2px 12px rgba(0,0,0,0.5);

📌 SDG11 進度條動畫原理：
1. HTML 裡用 data-value="55" 存儲數值
2. CSS 設 transition: width 1.3s cubic-bezier(...)
3. JS 用 IntersectionObserver 偵測到元素進入視窗時，設定 style.width = '55%'
4. width 從 0% 平滑動到 55%，看起來就像動畫
